# 任务摘要：屏幕流优化 - scrcpy 方案探索与替代方案实现

## 目标
优化设备屏幕实时推送延迟（原方案 3.3s/帧），实现流畅遥控器体验。

## 关键发现
1. **scrcpy-server 在该设备上不可行** — `app_process` 加载 DEX 失败
2. **adb exec-out 持久 shell 方案不可行** — ADB Windows 版对 exec-out shell 传参支持差
3. **adb shell + wm size 720p 方案是最优**：
   - 1080p：0.13 FPS，2.8MB/帧
   - 720p (`wm size 720x405`)：1.2 FPS，100KB/帧
   - 提升约 9 倍帧率
